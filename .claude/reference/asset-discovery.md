# Asset Discovery — MCP for assets, Argon for scripts

> Agents: read this once per task that touches Roblox instances. This is the single source of truth for where to look up paths.

---

## The split

| Target | Tool | Why |
|---|---|---|
| `StarterGui.*` (UI) | **Roblox Studio MCP** | Not Argon-synced — StarterGui has no `$path` |
| `Workspace.*` (parts, models, VFX) | **MCP** | Not Argon-synced |
| `ReplicatedStorage.Assets` | **MCP** | Not Argon-synced (undeclared in `default.project.json`) |
| `ServerStorage.Assets` | **MCP** | Not Argon-synced (sibling to `ServerUtilities` which IS synced) |
| Scripts / LocalScripts / ModuleScripts | **Argon** — use Read/Edit/Write on `src/` | Reliable for text files |
| `DataModules/*.luau` configs | **Argon** — Read/Edit/Write | Source-controlled code |
| `ReplicatedStorage.Packages.*` (Wally) | **Argon-adjacent** — read files under `Packages/` | Generated by `wally install` |

**Never edit a script via MCP.** Always use the local filesystem for code.
**Never guess an instance path from `src/StarterGui/` or `src/Workspace/`** — those are archived; the live tree is in Studio.

---

## Protocol for any task touching assets

1. **Read `filestructure/<root>.md`.** Cheap warm-up — pulls instance names into context.
2. **Verify before acting.** If you're about to write code that references a path, call `mcp__Roblox_Studio__inspect_instance(path: "...")` first. The MD is a cache; Studio is ground truth.
3. **Refresh the snapshot.** After `search_game_tree` or `inspect_instance` returns new or changed children, rewrite the matching `filestructure/*.md` and bump the entry in `filestructure/last-updated.json`. Skip refresh only if the tree matches.
4. **Probe MCP availability at session start.** If `mcp__Roblox_Studio__list_roblox_studios` returns an empty list, STOP. Tell the user Studio isn't connected. **Do not fabricate paths from stale MD.**

---

## Worked example — "Add a wins label to the top bar"

1. Read `filestructure/starter-gui.md` → confirm `MainGui.TopBar` exists.
2. `mcp__Roblox_Studio__inspect_instance(path: "StarterGui.MainGui.TopBar")` → confirm it has a `WinsLabel` child (or doesn't — if the user just added it, refresh).
3. **Edit the script** via Read/Edit/Write: `src/Client/Client/Modules/UIClient.luau` to wire the label.
4. If inspect_instance revealed a new child, rewrite `filestructure/starter-gui.md` and bump `last-updated.json.starter-gui`.
5. **Do NOT** touch the UI instance itself via `multi_edit` unless the user explicitly asked — instance edits belong to the user working in Studio, not the agent.

## Key MCP tools

| Tool | Use |
|---|---|
| `mcp__Roblox_Studio__list_roblox_studios` | Probe availability at task start. Returns empty → stop. |
| `mcp__Roblox_Studio__set_active_studio` | Select a specific Studio when multiple are open. |
| `mcp__Roblox_Studio__search_game_tree` | Initial discovery + bulk snapshot regeneration. Use `path` + `max_depth: 6`. |
| `mcp__Roblox_Studio__inspect_instance` | Per-path verification before acting. Returns properties + immediate children. |
| `mcp__Roblox_Studio__screen_capture` | When the user describes a visual bug and text doesn't cut it. |
| `mcp__Roblox_Studio__script_read` / `script_grep` / `script_search` | ONLY when the script under question was authored in Studio and not yet pulled into `src/`. Prefer Read/Grep on the local file. |
| `mcp__Roblox_Studio__multi_edit` | **Instance edits, not scripts.** Only when the user explicitly asks you to modify the live tree. |

---

## Corollary: what NOT to do

- Don't search `src/StarterGui/` (it's archived) for UI instance names.
- Don't assume a Workspace path exists because it was there "last time" — always verify.
- Don't skip the snapshot refresh after `search_game_tree` — stale MD misleads the next agent.
- Don't edit script content through `mcp__Roblox_Studio__multi_edit` — use Argon paths.
- Don't run `mcp__Roblox_Studio__execute_luau` for data mutations — that's a DataService/SignalBank concern on the real server, not a one-off REPL.
