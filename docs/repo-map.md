# Repo map

Folder-by-folder tour of the boilerplate. Read this before major feature work or unfamiliar bug investigation.

```

├── default.project.json   ← Argon mapping: folder → Studio DataModel

├── wally.lock             ← (gitignored) generated by `wally install`

├── stylua.toml            ← formatter config (100 col, tab, AutoPreferDouble)

├── docs/                  ← human documentation (architecture, cookbook, conventions, repo-map)

```

## `StarterGui` (in-Studio only — NOT under `src/`)

StarterGui is no longer Argon-synced. Build the `Ui` ScreenGui and its `Frames` folder directly in Studio.

Convention: every visible frame lives at `StarterGui.Ui.Frames.<FrameName>`. Client code accesses them via `UIClient.GetFrame("FrameName")`.

Discovery:
- Read `filestructure/starter-gui.md` for the committed tree snapshot.
- Verify live paths with `mcp__Roblox_Studio__inspect_instance(path: "StarterGui.Ui.Frames.<name>")`.
- After adding/removing instances in Studio, refresh `filestructure/starter-gui.md` and bump `filestructure/last-updated.json`.

The previous scaffolding under `src/StarterGui/` has been moved to `src/_archive/StarterGui/` for reference.

## `Packages/` and `src/Server/Server/Packages/` (gitignored)

These are generated by `wally install` and should NEVER be committed. `Packages/` (at repo root) mounts into Studio as `ReplicatedStorage.Packages`. Server packages live under `src/Server/Server/Packages/` and mount as `ServerScriptService.Server.Packages` via `default.project.json`'s `$path: "src/Server"`.

Wally's `[server-dependencies]` write to the repo-root `ServerPackages/` folder by default — after each `wally install`, run:

    mv ServerPackages/* src/Server/Server/Packages/ && rmdir ServerPackages

If you delete these folders, re-run `wally install` (plus the move above) to regenerate.

## Convention summary

- Handlers live under `Server/Modules/` and must be registered in `GameService.luau`.
- Client modules live under `Client/Modules/` and must be registered in `GameController.luau`.
- All bridges live in `Shared/Bridges.luau` — never create remotes anywhere else.
- All dev product IDs and balance values live in `Shared/DataModules/` — never hardcode them in handlers.
- Persistent data schema lives in `DataService.luau`'s `DATA_TEMPLATE`.
- DataStore key `BoilerplateData_LIVE_1` never changes (rename it ONCE when you fork the boilerplate, then never again).

See `docs/conventions.md` for the full style guide and ban list.
